# 知识库接口文档 v2.4

## 说明

基于当前 v2.4 工程实际 controller、request DTO、response DTO 整理。

统一前缀：`/api`

统一错误返回：

```json
{
  "error": "BAD_REQUEST",
  "message": "具体错误信息"
}
```

或：

```json
{
  "error": "INTERNAL_SERVER_ERROR",
  "message": "具体错误信息"
}
```

---

## 1. 文档导入与查看

### 1.1 导入文档

**POST** `/api/documents/import`

请求体：

```json
{
  "documentType": "MARKDOWN",
  "title": "示例文档",
  "sourceUri": "file:///tmp/demo.md",
  "content": "# 标题\n正文内容",
  "ontologyKey": "RELOC"
}
```

字段说明：
- `documentType`：必填，支持 `PLAIN_TEXT` / `MARKDOWN` / `HTML`
- `title`：必填，文档标题
- `sourceUri`：可选，来源地址
- `content`：必填，文档正文
- `ontologyKey`：可选，导入后立即执行 segment -> ontology 挂接

返回：

```json
{
  "documentId": "DOC_xxx",
  "segmentCount": 18,
  "ontologyLinkCount": 6,
  "status": "IMPORTED"
}
```

---

### 1.2 重建某文档的 ontology 挂接

**POST** `/api/documents/{documentId}/ontology-links/rebuild?ontologyKey={ontologyKey}`

示例：

`/api/documents/DOC_xxx/ontology-links/rebuild?ontologyKey=RELOC`

返回：

```json
{
  "documentId": "DOC_xxx",
  "ontologyKey": "RELOC",
  "ontologyLinkCount": 12,
  "status": "RELINKED"
}
```

---

### 1.3 查询文档列表

**GET** `/api/documents`

返回：

```json
[
  {
    "documentId": "DOC_xxx",
    "documentType": "MARKDOWN",
    "title": "示例文档",
    "status": "IMPORTED",
    "createdAt": "2026-04-21 10:00:00"
  }
]
```

---

### 1.4 查询某文档的切片列表

**GET** `/api/documents/{documentId}/segments`

返回：

```json
[
  {
    "segmentId": "SEG_xxx",
    "title": "步骤一",
    "content": "正文内容",
    "segmentType": "PARAGRAPH",
    "path": "/1/2",
    "orderNo": 12
  }
]
```

## 2. 本体导入与查看

### 2.1 导入本体

**POST** `/api/ontology/import`

请求体：

```json
{
  "ontologyKey": "RELOC",
  "ontologyName": "机房搬迁本体",
  "version": "2.4",
  "description": "示例本体",
  "replaceExisting": true,
  "types": [
    {
      "typeKey": "method.engineering",
      "typeName": "工程操作方法",
      "description": "方法类型",
      "parentTypeKey": null,
      "sortNo": 1
    }
  ],
  "nodes": [
    {
      "nodeKey": "method.engineering.ring_insertion",
      "name": "环插方法",
      "typeKey": "method.engineering",
      "description": "环插操作方法",
      "aliases": ["光层环插"]
    }
  ],
  "edges": [
    {
      "sourceNodeKey": "method.engineering.ring_insertion",
      "targetNodeKey": "condition.selection.ring_insertion_feasible",
      "relationType": "requires",
      "description": "需要满足条件",
      "bidirectional": false
    }
  ]
}
```

返回：

```json
{
  "ontologyId": "OG_xxx",
  "ontologyKey": "RELOC",
  "typeCount": 8,
  "nodeCount": 42,
  "aliasCount": 17,
  "edgeCount": 56,
  "status": "IMPORTED"
}
```

说明：
- `replaceExisting=true` 时，若同 key 本体已存在，会覆盖旧版本。

## 4. 检索接口

### 4.1 旧版文本检索

**POST** `/api/search`

请求体：

```json
{
  "query": "环插",
  "topK": 10
}
```

说明：
- `topK` 可选，默认 10

返回：

```json
{
  "query": "环插",
  "hits": [
    {
      "segmentId": "SEG_xxx",
      "documentId": "DOC_xxx",
      "title": "步骤一",
      "content": "进行光层环插",
      "score": 10.8,
      "orderNo": 12,
      "previousSegmentTitle": "前一段内容",
      "nextSegmentTitle": "后一段内容"
    }
  ]
}
```

## 7. 常见调用顺序

### 7.1 新文档接入
1. 导入 ontology：`POST /api/ontology/import`
2. 导入 document：`POST /api/documents/import`
3. 必要时重建索引：
   - `POST /api/indexes/document/{documentId}/rebuild`
   - `POST /api/vector-index/document/{documentId}/rebuild`
4. 检索：`POST /api/retrieval/search`

### 7.2 老数据补索引
1. `POST /api/indexes/ontology/{ontologyKey}/rebuild`
2. `POST /api/indexes/document/{documentId}/rebuild`
3. `POST /api/vector-index/ontology/{ontologyKey}/rebuild`
4. `POST /api/vector-index/document/{documentId}/rebuild`
